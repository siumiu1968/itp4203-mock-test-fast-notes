Lab 4：Android Studio 加 Room API 教程

呢段係根據 `Mobile Lab 4.docx` Step 2 至 Step 13 整理。考試如果要用 Room，先做 Gradle 設定，之後先寫 Entity、Dao、Database。

核心理解

Room 即係 Android 入面一個「幫你管理 SQLite database 嘅助手」。你唔需要自己寫太多低層 SQL 開 database，主要寫三樣：`@Entity` = 一張表、`@Dao` = 查/加/改/刪方法、`@Database` = database 主入口。

KSP 即係「幫 Room 自動生 code 嘅工具」。如果唔加 KSP，Android Studio 會唔識幫你把 `@Dao`、`@Database` 轉成真正可用嘅 database code。

Step 2

開 `Gradle Scripts -> libs.versions.toml`

用途：先定義 Room、SQLite、KSP 嘅版本同 library 名。之後其他 Gradle 檔就可以用 `libs.androidx.room.runtime` 呢啲短名。

貼入對應 block，唔好重覆整多個 `[versions]`。 
[versions]
room = "2.8.4"
sqlite = "2.6.2"
ksp = "2.3.3"

[libraries]
androidx-room-runtime = { module = "androidx.room:room-runtime", version.ref = "room" }
androidx-room-compiler = { module = "androidx.room:room-compiler", version.ref = "room" }
androidx-sqlite-bundled = { module = "androidx.sqlite:sqlite-bundled", version.ref = "sqlite" }

[plugins]
kotlin-android = { id = "org.jetbrains.kotlin.android", version.ref = "kotlin" }
ksp = { id = "com.google.devtools.ksp", version.ref = "ksp" }

room = "2.8.4"：Room library 版本。

ksp = "2.3.3"：KSP plugin 版本。

androidx-room-runtime：app 執行時真正用 Room。

androidx-room-compiler：KSP 用佢幫你生 database code。

Step 3

開 Project `build.gradle.kts`

用途：喺成個 project 宣告「我呢個 project 有 KSP 呢個 plugin 可以用」。貼完按 `Sync Now`。

位置：`plugins { ... }` 入面。 
plugins {
    alias(libs.plugins.android.application) apply false
    alias(libs.plugins.kotlin.android) apply false
    alias(libs.plugins.ksp) apply false
}

如果你本身個 project 已經有其他 plugin，唔好刪走；只係加返 `alias(libs.plugins.ksp) apply false`。

Step 4

開 Module `build.gradle.kts (:app)`

用途：真正喺 app module 啟用 KSP。冇呢句，Room annotation processor 唔會幫你做嘢。

安全寫法：保留 Kotlin plugin，再加 KSP。 
plugins {
    alias(libs.plugins.android.application)
    alias(libs.plugins.kotlin.android)
    alias(libs.plugins.ksp)
}

Lab 圖入面重點係加 `alias(libs.plugins.ksp)`。如果你刪走 `alias(libs.plugins.kotlin.android)`，Kotlin project 可能會 compile 唔到。

Step 5

同一個 `:app` 檔案入 dependencies

用途：正式把 Room library 加入 app。`implementation` 係 app 用，`ksp` 係編譯時幫你生 code，`lifecycle-viewmodel-ktx` 係 Lab 4 後面 `TaskViewModel` 會用。

位置：`dependencies { ... }` 入面。 
dependencies {
    implementation(libs.androidx.room.runtime)
    ksp(libs.androidx.room.compiler)
    annotationProcessor(libs.androidx.room.compiler)
    implementation("androidx.lifecycle:lifecycle-viewmodel-ktx:2.10.0")
}

implementation(...room.runtime)：app 跑嘅時候要用 Room。

ksp(...room.compiler)：編譯時幫你處理 `@Entity`、`@Dao`、`@Database`。

annotationProcessor(...)：Lab 4 有叫你加；Kotlin 主要靠 KSP。

lifecycle-viewmodel-ktx：令 `viewModelScope.launch` 呢類寫法可用。

Step 6

如果 AGP 9.0 或以上，開 `gradle.properties`

用途：Lab 4 話如果 Android Gradle Plugin 比較新，可能要加呢句，避免某啲舊式 Kotlin sourceSets plugin 出問題。

如果你 AGP 舊過 9.0，Lab 話可以唔做。 
android.disallowKotlinSourceSets=false

做完 Gradle 後一定要做

  1. 撳 Android Studio 右上角 `Sync Now`。
  2. 等下面 Gradle sync 完，冇紅字先繼續。
  3. 如果 `@Entity`、`@Dao`、`RoomDatabase` 係紅色，多數係 dependencies 未 sync 或 import 未加。
  4. 如果 `ksp(...)` 紅色，多數係 Project Gradle 未加 `alias(libs.plugins.ksp) apply false`，或者 app plugin 未加 `alias(libs.plugins.ksp)`。
  5. 如果見到 `Inconsistent JVM Target Compatibility`，喺 Module `build.gradle.kts` 加下面 JVM 17 設定。

Android Studio 2025.2 常見修正：JVM target mismatch

如果 build error 寫 `compileDebugJavaWithJavac (1.8)` 同 `compileDebugKotlin (21)` 唔一致，就將呢段放入 `app/build.gradle.kts` 嘅 `android { ... }` 後面。

android {
    compileOptions {
        sourceCompatibility = JavaVersion.VERSION_17
        targetCompatibility = JavaVersion.VERSION_17
    }
}

kotlin {
    compilerOptions {
        jvmTarget.set(org.jetbrains.kotlin.gradle.dsl.JvmTarget.JVM_17)
    }
}

import androidx.room.Dao
import androidx.room.Delete
import androidx.room.Insert
import androidx.room.OnConflictStrategy
import androidx.room.Query
import androidx.room.Update
import kotlinx.coroutines.flow.Flow

@Dao
interface TaskDao {

    fun getAllTasks(): Flow<List<Task>>

    @Insert(onConflict = OnConflictStrategy.REPLACE)
    suspend fun insertTask(task: Task)

    @Update
    suspend fun updateTask(task: Task)

    @Delete
    suspend fun deleteTask(task: Task)
}

@Dao：話俾 Room 知呢個 interface 係 database 操作清單。

@Query(...)：用 SQL 讀資料，並按 priority 由細至大排序。

Flow<List<Task>>：資料有變時，畫面可以自動收到新 list。

suspend：database 操作可以放入 coroutine，避免阻塞畫面。

3. `AppDatabase.kt`：Database 主入口

import android.content.Context
import androidx.room.Database
import androidx.room.Room
import androidx.room.RoomDatabase

@Database(entities = [Task::class], version = 1, exportSchema = false)
abstract class AppDatabase : RoomDatabase() {
    abstract fun taskDao(): TaskDao

    companion object {
        @Volatile
        private var INSTANCE: AppDatabase? = null

        fun getDatabase(context: Context): AppDatabase {
            return INSTANCE ?: synchronized(this) {
                val instance = Room.databaseBuilder(
                    context.applicationContext,
                    AppDatabase::class.java,
                    "task_database"
                ).build()

                INSTANCE = instance
                instance
            }
        }
    }
}

@Database(...)：話俾 Room 呢個 database 有邊啲表。

abstract fun taskDao()：提供一個方法拎 `TaskDao`。

@Volatile：確保多條 thread 都睇到同一個 INSTANCE。

Room.databaseBuilder(...)：真正建立 `task_database` 呢個 database。

實測發現：最容易卡住嘅位

我用 Android Studio 2025.2 / Kotlin / Room 實際 build 同 emulator 跑過一次，下面係真實遇到或確認過嘅問題。

1. Project 放外置碟可能會 build fail

錯誤樣：`._layout is not a directory`、或者 resource parser 指住 `._xxx` 檔案。

原因：macOS 喺某啲外置碟格式會自動產生 AppleDouble 檔案，例如 `._layout`、`._activity_main.xml`。Android resource parser 會當佢係壞 resource。

即時做法：考試時把 project 放喺 `Desktop` / `Documents` / Android Studio 預設內置硬碟位置，唔好喺 USB / 外置 SSD 入面 build。

如果已經中咗：關 Android Studio，刪走所有 `._*` 檔，最好搬 project 去內置硬碟再 Sync / Run。

2. Kotlin / Java JVM target 唔一致

錯誤樣：`Inconsistent JVM-target compatibility detected for tasks compileDebugJavaWithJavac (1.8) and compileDebugKotlin (21)`。

原因：Android Studio 2025.2 / Kotlin 2.2 可能用較新 JVM target，但 Android Java compile option 仍係舊設定。

即時做法：去 JVM target mismatch 修正 copy JVM 17 設定，放入 `app/build.gradle.kts`。

加完要 `Sync Now`，再 Run。

3. Save 按鈕好似冇反應

現象：輸入 Name / Description 後按 Save，畫面仍停喺 Insert page。

原因：多數係 keyboard / EditText focus 仍然食緊第一下操作，特別係 emulator 或細螢幕。

即時做法：先按一次 Back 收起 keyboard，再按 Save；或者點畫面空白位後再 Save。

我實測：收 keyboard 後再按 Save，Room insert 成功，首頁即時顯示新資料。

4. Select Image 會開系統 picker，冇圖都算成功一半

現象：按 Select Image 後打開 `Recent images` / `Photos` / `Drive` picker，但 emulator 可能顯示 `No items`。

原因：模擬器入面未必有圖片。考試機有圖片就可以揀；冇圖片時至少證明 picker 開到。

即時做法：如果時間唔夠，可以先確保 Name 可以 Save；有圖先測 image preview。

評分重點通常係有 Select Image button、能開 picker、揀到後能 `setImageURI`。

        val tvName: TextView = itemView.findViewById(R.id.tvName)

package com.example.mocktestanswer

import android.net.Uri
import android.os.Bundle
import android.widget.EditText
import android.widget.ImageView
import android.widget.Toast
import androidx.activity.result.contract.ActivityResultContracts
import androidx.appcompat.app.AppCompatActivity

class InsertActivity : AppCompatActivity() {

    private lateinit var db: AppDatabase
    private lateinit var etName: EditText
    private lateinit var etDescription: EditText
    private lateinit var imageView: ImageView
    private lateinit var btnSelectImage: MaterialButton
    private lateinit var btnCancel: MaterialButton
    private lateinit var btnSave: MaterialButton
    private var selectedImageUri: Uri? = null

    // Open the system image picker and preview the selected image.
    private val imagePicker =
        registerForActivityResult(ActivityResultContracts.GetContent()) { uri ->
            if (uri != null) {
                selectedImageUri = uri
                imageView.setImageURI(uri)
                imageView.visibility = ImageView.VISIBLE
            }
        }

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_insert)

        db = AppDatabase.getDatabase(this)

        etName = findViewById(R.id.etName)
        etDescription = findViewById(R.id.etDescription)
        imageView = findViewById(R.id.imageView)
        btnSelectImage = findViewById(R.id.btnSelectImage)
        btnCancel = findViewById(R.id.btnCancel)
        btnSave = findViewById(R.id.btnSave)

        btnSelectImage.setOnClickListener {
            imagePicker.launch("image/*")
        }

        btnCancel.setOnClickListener {
            finish()
        }

        btnSave.setOnClickListener {
            saveCamera()
        }
    }

    // Save a new camera record and return to the start page.
    private fun saveCamera() {
        val name = etName.text.toString().trim()
        val description = etDescription.text.toString().trim()

        if (name.isEmpty()) {
            Toast.makeText(this, "Please enter camera name", Toast.LENGTH_SHORT).show()
            return
        }

        val camera = Camera(
            name = name,
            descriptions = description,
            imagePath = selectedImageUri?.toString() ?: ""
        )

        db.cameraDao().insertCamera(camera)
        finish()
    }
}

ActivityResultContracts.GetContent()：叫 Android 開系統選圖器。

selectedImageUri?.toString() ?: ""：有圖就存 URI，冇圖就存空字串。

Toast.makeText(...)：Name 空白時提示使用者。

finish()：Save 完關閉 Insert page，回到首頁。
